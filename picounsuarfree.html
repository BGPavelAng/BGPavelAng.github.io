Esta prueba es relativamente sencilla, casí nos dan todo, solo para ejecutar la vulnerabilidad.

Abrimos el programa y observamos que es una especie de acceso y registro.

Al oprimir s, nos dice "OOP! Memory leak...0x80487d6".

El problema es, ¿para qué funciona?.

Empezamos a meter pattern de datos en los inputs, pero nada sucede hasta que observemos algo raro. Cuando creamos un usuario, nos dice que podemos mandar un mensaje.

Pero cuando nos deslogueamos y tratamos de mandar un mensaje nos marca un error.

Ok, ya tenemos una dirección de memoria y un error, ahora... ¿cómo están relacionados y qué podemos hacer?

Bueno, recordemos una vez más, que malloc crea bloques de memoria ya sea dentro de heap, chunks fuera de heap o crea arenas cuando es demasido el espacio de memoria requerida.

Al ser muy poco el espacio, todo debe de estár en heap, y memory leak es la dirección de hahaexploitgobrrr donde se encuentra el read de flag.

Ok, eso siginifica que malloc crea un usuario abriendo un bloque en heap para guardar el nombre.

Ahora vamos a desloguearnos y veamos que pasa..

Podemos ver que no fue borrado, solo fue liberado para su reutilización.
